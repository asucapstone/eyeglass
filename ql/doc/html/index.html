Quick Link 2

Introduction

Quick Link 2 is an API which allows developers to easily add eye tracking directly within their own application.


Files

Quick Link 2 has three main files that are needed for using the API.

  • QLTypes.h - This contains the definitions of all the structures, types, and enumerations used in Quick Link 2.
  • QuickLink2.h - This contains the declarations of all the functions available in Quick Link 2.
  • QuickLink2.dll - This is the dynamically linked library containing the definitions of all the available functions.

There are some additional files that are not required, but are available to aid in development.

  • QuickLink2_ptr_C.h - This contains c-style declarations of function pointers for all available functions. These function pointers can aid in explicit linking.
  • QuickLink2.lib - This is a linker file which is used for implicit linking of the API.

There are some other DLL files that are included. They are device specific files and are loaded explicitly by QuickLink2.dll. These files handle the communication with the devices. QuickLink2.dll will load and run correctly without them but its usefulness would be very limited because it would not be able to find any devices attached to the computer.

  • PGRFlyCapture.dll
  • SMX11MX.dll

For redistribution of an application that uses Quick Link 2 only three files need to be included:

  • QuickLink2.dll
  • PGRFlyCapture.dll
  • SMX11MX.dll


FileStructure

There are eight folders at the top level of the install directory for Quick Link 2:

  • bin - This contains all the 32-bit binary files needed to use the API and run the sample project.
  • bin64 - This contains all the 64-bit binary files needed to use the API and run the sample project.
  • doc - This contains the documentarian for Quick Link 2.
  • include - This contains the header files needed for Quick Link 2.
  • lib - This contains the 32-bit linker files for Quick Link 2.
  • lib64 - This contains the 64-bit linker files for Quick Link 2.
  • OpenCV2.2 - This contains the OpenCV files needed for the sample project to compile and run.
  • src - This contains the sample project for Quick Link 2.


Function Layout

Each function in Quick Link 2 is categorized into one of four different sections:

  • API - Functions that affect the API as a whole.
  • Device - Functions that affect devices.
  • Settings - Functions that affect settings containers.
  • Calibration - Functions that affect calibration containers.


Examples

There is an example project in the /src/QuickStart/ folder. The project contains a preconfigured project file for visual studio 2008 which should allow the project to quickly be built and run. The output files are placed in the /bin or /bin64 folder.The main files that contain code for the project are:

  • Calibrate.cpp - This contains the process for calibrating.
  • DisplayVideo.cpp - This contains the process for getting frames from a device and displaying them.
  • Initialize.cpp - This contains the process for getting a device ready to use.
  • Main.cpp - This contains the process for starting the device and getting frame data.
  • OpencvUtility.cpp - This contains common drawing functions used by functions in other files.
  • QL2Utility.cpp - This contains a small class that helps with explicit loading and unloading of the API.

The sample project relies on OpenCV for displaying graphical output. The OpenCV header files and linker files are in the OpenCV2.2 folder. The runtime DLLs are in the /bin and /bin64 folders.

Quick Link 2 does not use OpenCV so the OpenCV DLLs in the /bin and /bin64 folders are only there for the purpose of running the sample application.
